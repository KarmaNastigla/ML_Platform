# 🚀 Universal ML Experimentation Platform

**Платформа полного цикла машинного обучения:** загрузи датасет, очисти данные, обучи модель, разберись почему она так решает — и задеплой. Всё в одном интерфейсе, без написания кода.

## Зачем этот проект

Большинство ML-инструментов решают одну часть задачи: MLflow — трекинг экспериментов, AutoSklearn — подбор модели, FastAPI — деплой. Чтобы связать их вместе, нужно самому писать склейку.

Эта платформа закрывает весь цикл в единый UX: от сырого CSV до задеплоенного API — без переключения инструментов. При этом под каждым решением видна логика: какие признаки важны, почему модель предсказала именно так, насколько стабильна оценка качества.

## Возможности

### 📊 Анализ и очистка данных
- Автоматический отчёт о качестве: пропуски, выбросы, типы данных для каждого признака
- Удаление столбцов по настраиваемому порогу пропусков — с превью что будет удалено
- Сглаживание выбросов с настраиваемым множителем IQR (по умолчанию 1.5, рекомендация прямо в UI)
- Заполнение пропусков на уровне каждого столбца: Медиана / Среднее / Мода / Константа
- Создание новых признаков по математической формуле: `(SibSp + Parch) * Fare`
- Интерактивные графики: корреляционная матрица, гистограммы, scatter, boxplot
- Счётчик изменений датасета (Δ строк / Δ столбцов от исходного)

### ⚙️ Обучение модели
- **Автоопределение типа задачи** — `float`-таргет или >20 уникальных значений → регрессия; иначе → классификация. Исключает случайный запуск классификационных метрик на регрессии
- **Bayesian оптимизация гиперпараметров** через Optuna — находит хорошие параметры за 20–50 итераций вместо полного перебора GridSearch
- **K-Fold Cross-Validation** — опциональный режим с настраиваемым числом фолдов; финальная модель обучается на всех данных
- **4 алгоритма:** Random Forest, Gradient Boosting, Logistic Regression / Ridge, Voting Ensemble (RF + GB + LR/Ridge)

### 🔍 Объяснимость (XAI)
- **SHAP Waterfall** — для любого предсказания: какой признак на сколько сдвинул результат от базового уровня. Встроен прямо в тестер — не нужно создавать отдельный CSV
- **График важности признаков** — топ-15 по impurity decrease (деревья) или абсолютному весу коэффициента (линейные модели)
- **Confusion Matrix** — с нормировкой по строкам и двойной аннотацией: абсолютные числа + процент по классу

### 🧪 Встроенный тестер
- Форма с умными виджетами: `selectbox` для категориальных признаков и числовых с небольшим числом уникальных значений (например, Pclass → `[1, 2, 3]`), `number_input` для непрерывных
- Дефолтные значения — медианы из обучающей выборки
- SHAP-объяснение строится автоматически для введённых данных сразу после предсказания
- Для классификации — bar chart вероятностей по всем классам

### 📜 История экспериментов
- Журнал всех запусков: модель, таргет, тип CV, метрики, время
- Визуальное сравнение любой метрики по всем экспериментам на одном графике
- Скачать историю в CSV
- **Генератор воспроизводимого `.py`-скрипта** — подробнее ниже

### 🛠 Деплой
- One-click генерация FastAPI-сервиса + Dockerfile + docker-compose.yml
- Продакшен-образ ~300 МБ (dev-образ с Streamlit/Optuna/SHAP весил бы ~2 ГБ)

## Быстрый старт

```bash
# 1. Клонируй репозиторий
git clone https://github.com/KarmaNastigla/ML_Platform
cd ML_Platform

# 2. Создай виртуальное окружение (опционально)
python -m venv .venv
source .venv/bin/activate       # Linux / macOS
.venv\Scripts\activate          # Windows

# 3. Установи зависимости
pip install -r requirements.txt

# 4. Запусти платформу
streamlit run app.py
```

Браузер откроется на `http://localhost:8501`. Загрузи любой CSV-файл в боковое меню — и начинай.

## Деплой модели

После обучения нажми **"🛠 Сгенерировать API и Dockerfile"** — платформа создаст папку `deploy/`.

**Запуск:**
```bash
cd deploy
docker-compose up --build
# API доступен на http://localhost:8000
```

**Эндпоинты:**

| Метод | Путь | Что возвращает |
|-------|------|----------------|
| `GET` | `/health` | Статус, тип задачи, список признаков |
| `POST` | `/predict` | Предсказание для переданных данных |

**Пример запроса:**
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{"data": {"Pclass": 1, "Sex": "female", "Age": 28, "Fare": 100}}'
# → {"prediction": 1, "task_type": "classification"}
```

**Почему образ ~300 МБ, а не ~2 ГБ:**
`deploy_generator.py` задаёт `PRODUCTION_REQUIREMENTS` как константу — только `fastapi`, `uvicorn`, `scikit-learn`, `pandas`, `joblib`. `deploy/requirements.txt` никогда не копируется из корневого, чтобы Streamlit, Optuna и SHAP не попали в образ.

## Экспорт воспроизводимого Python-скрипта

Во вкладке **"📜 История экспериментов"** выбери любой запуск и нажми **"⬇️ Сгенерировать и скачать .py скрипт"**.

Скрипт содержит весь пайплайн от начала до конца:

```
1. Загрузка данных
2. Разведочный анализ (EDA)    ← describe, value_counts, корреляции, выбросы IQR
3. Очистка данных              ← только шаги, которые ты реально применил в UI
4. Подготовка признаков        ← ColumnTransformer: импутация, скейлинг, кодирование
5. Обучение модели             ← с точными параметрами Optuna, без повторного поиска
6. Оценка                      ← метрики + confusion matrix
7. Сохранение модели           ← joblib.dump()
```

Раздел «3. Очистка данных» появляется только если ты реально нажимал кнопки очистки — иначе нумерация сдвигается и раздела нет. Параметры модели вшиты напрямую (`n_estimators=120, learning_rate=0.066`), поэтому Optuna при запуске скрипта не нужен.

```bash
python ml_solution_Gradient_Boosting_Survived.py
```

## Поддерживаемые алгоритмы

| Алгоритм | Задача | Оптимизируемые параметры |
|----------|--------|--------------------------|
| Random Forest | классификация / регрессия | `n_estimators`, `max_depth` |

## Технические решения

### Optuna вместо GridSearch
GridSearch с 3 параметрами по 10 значений = 1000 обучений. Optuna (TPE Sampler) — байесовская оптимизация: каждый следующий trial учитывает предыдущие результаты. На Titanic это разница между ~2 минутами и ~20 секундами при сопоставимом качестве.

### K-Fold CV внутри Optuna
При включённом CV каждый trial оценивается на K фолдах — исключает везение с конкретным split. Финальная модель обучается на всех данных, что стандартно для производственного использования.

### SHAP: TreeExplainer для деревьев, LinearExplainer для линейных
`TreeExplainer` использует внутреннюю структуру дерева — точный и быстрый. Универсальный `KernelExplainer` перебирает подмножества признаков — работает минутами. Выбор explainer'а автоматический в зависимости от типа модели.

### Soft Voting в ансамбле
`voting='soft'` усредняет вероятности моделей, а не голосует за метку. Работает лучше когда модели по-разному уверены в разных наблюдениях.

### Лог очистки для воспроизводимости
Каждое нажатие кнопки в UI записывается в `cleaning_log` (`session_state`). При запуске обучения лог фиксируется вместе с параметрами Optuna. Генератор скрипта переводит лог в исполняемый Python-код — шаг за шагом, в том порядке, в котором они применялись.
