## Что умеет платформа

| Блок | Возможности |
|------|------------|
| **EDA и очистка** | Отчёт о качестве данных, удаление столбцов по порогу, сглаживание выбросов (настраиваемый IQR), гибкое заполнение пропусков по столбцам, feature engineering по формуле |
| **Автообучение** | Автоопределение типа задачи (классификация / регрессия), Bayesian оптимизация гиперпараметров через Optuna, поддержка K-Fold CV |
| **Объяснимость (XAI)** | SHAP Waterfall для каждого предсказания, bar chart важности признаков, confusion matrix с нормировкой по строкам |
| **Тестер** | Встроенный тестер с умными виджетами (selectbox / number_input по типу признака), SHAP автоматически для введённых данных |
| **История** | Журнал всех экспериментов, визуальное сравнение метрик, генерация воспроизводимого Python-скрипта для выбранного запуска |

## Быстрый старт

```bash
# 1. Клонируй репозиторий
git clone https://github.com/your-username/universal-ml-platform.git
cd universal-ml-platform

# 2. Создай виртуальное окружение (опционально)
python -m venv .venv
source .venv/bin/activate       # Linux / macOS
.venv\Scripts\activate          # Windows

# 3. Установи зависимости
pip install -r requirements.txt

# 4. Запусти платформу
streamlit run app.py
```

Браузер откроется автоматически на `http://localhost:8501`.

## Технические решения и почему они приняты

### Автоматическое определение типа задачи
Система анализирует целевую колонку: `float`-тип или более 20 уникальных значений → регрессия, иначе → классификация. Это исключает ошибочный запуск классификационных метрик на регрессионной задаче.

### Optuna вместо GridSearch
GridSearch перебирает все комбинации в декартовом произведении — при 3 параметрах по 10 значений это 1000 обучений. Optuna использует байесовскую оптимизацию (TPE Sampler): каждый следующий trial опирается на результаты предыдущих, находя хорошие параметры за 20–50 итераций. На датасете Titanic это разница между 2 минутами и 20 секундами.

### K-Fold CV внутри Optuna
При включённом CV каждый trial оценивается на K фолдах, что исключает случайное везение с конкретным split. Финальная модель обучается на **всех** данных — это стандартная практика, позволяющая использовать максимум доступной информации.

### SHAP TreeExplainer vs KernelExplainer
Для деревьев используется `TreeExplainer` — он точный и работает за O(TLD²) вместо экспоненциального перебора подмножеств. `KernelExplainer` — универсальный, но медленный (минуты vs секунды). Для линейных моделей — `LinearExplainer` (аналитическое решение).

### Soft Voting в ансамбле
Ансамбль из RF + GradientBoosting + LogReg использует `voting='soft'`: финальный класс определяется по средней вероятности, а не голосованием за метку. Это работает лучше когда модели уверены с разной степенью.

### Лог очистки → воспроизводимый скрипт
Каждое нажатие кнопки в инструментах очистки записывается в `cleaning_log` в `session_state`. При сохранении эксперимента лог записывается вместе с параметрами модели. Генератор скрипта воспроизводит полный пайплайн — от загрузки CSV до `joblib.dump()`.

## Поддерживаемые алгоритмы

| Алгоритм | Задача | Оптимизируемые параметры |
|----------|--------|--------------------------|
| Random Forest | классификация / регрессия | `n_estimators`, `max_depth` |
| Gradient Boosting | классификация / регрессия | `n_estimators`, `learning_rate`, `max_depth` |
| Logistic Regression / Ridge | классификация / регрессия | `C` / `alpha` |
| Voting Ensemble | классификация / регрессия | фиксированные (RF + GB + LR) |

## Метрики

**Классификация:** Accuracy, Precision (macro), Recall (macro), Confusion Matrix, CV Accuracy ± std  
**Регрессия:** R², MAE, RMSE, CV R² ± std

## Тесты

```bash
pytest tests/ -v
```

Покрываются: автоопределение типа задачи, обучение классификаторов и регрессоров, SHAP для всех типов моделей, CV-режим, сохранение и загрузка модели.
